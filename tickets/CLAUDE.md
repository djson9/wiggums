# Tickets
This directory contains tickets / user stories that need to be resolved.

## Example
Example of what a ticket should look like.
<example>
---
Date: 2026-02-01
Title: ScriptPopup p.program is nil in Headless Mode
Status: completed
Description: User should be able to perform user desired action. We should verify that it works after implementing.
---
Did x, y, z actions and was 80% successful and 20% failed cases. << ALWAYS INCLUDE FAILED CASES
</example>

## Adding New Tickets
Add new tickets to this directory, with the filename format.
[EPOCH SECONDS]\_[Title].md
1769879447_Title.md

^^ Any tickets we pick up without this filename, we should update to standardize.
